# HS-01 Human-Validity Study Application — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the web application that serves the frozen 93-item HS-01 stimulus pool to ~30 lay raters across a three-phase validity study (text / image / pair) and collects a comprehensive, schema-conformant session record per participant.

**Architecture:** Next.js 14 (App Router; API routes are the backend — no separate service). Read-only inputs (frozen item pool + a generated study-config + consent text) are validated against their JSON Schemas at startup; the server refuses to boot on a pool-hash mismatch. One JSON file per participant session under a `data/sessions/` volume, written atomically and validated with ajv. Single Docker container (Next standalone output). The data-collection instrumentation (monotonic timing, render-complete onset, seeded randomization logging, semantic response coding, integrity events, stimulus-fidelity checks) is implemented exactly to `schemas/hs01.session.schema.json`.

**Tech Stack:** Next.js 14.2.x (App Router), React 18, TypeScript 5, Tailwind CSS 3.4, framer-motion 11, lucide-react, ajv 8 (draft 2020-12) + ajv-formats, vitest (unit), @playwright/test (e2e). Config generator + tests in Python via `conda run -n uni` (repo standard).

## Global Constraints

- **App location:** `experiments/HS-01/app/` (co-located with the pool, schemas, and `TOOL-DESIGN.md` it depends on). Reads `../pool_frozen/itempool.json` + `../pool_frozen/assets/images/` and `../schemas/`.
- **Authoritative spec:** `experiments/HS-01/TOOL-DESIGN.md` + the three schemas in `experiments/HS-01/schemas/`. Where this plan and a schema disagree, the schema governs — flag it.
- **Three schemas are binding contracts** (do not edit them): `hs01.itempool.schema.json` (input pool), `hs01.study-config.schema.json` (input config), `hs01.session.schema.json` (output record). Validate against all three with ajv draft 2020-12.
- **Stimulus fidelity (hard rules):** prompts rendered byte-identically — inserted as text nodes, never through any normalizing pipeline; font stack pinned; images served at native resolution, **no upscaling** (`css_w/h` must equal `natural_w/h`, asserted per trial via `rendered_image`).
- **Response coding is semantic, never positional:** pair responses stored as `ANCHOR_WORD | TARGET_WORD | OTHER_CLASS | NOTHING_RECOGNIZABLE | CANT_TELL`; the realized on-screen order is logged separately in `presented.option_display_order`.
- **Timing model:** all `*_ms` are monotonic offsets from session start via `performance.now()`; across reloads chain epochs (`offset = (load_epoch_utc − started_at_utc) + perf_delta`). Trial onset = stimulus fully rendered (`img.decode()` + next animation frame), never navigation. Reaction times are NOT stored (derived in analysis).
- **Privacy:** anonymous `participant_code` (`P###`) only; no IPs, no cookies (localStorage only); consent text versioned with a sha256.
- **Determinism / reproducibility:** session `rng_seed` is stored; with the config it reproduces trial order and per-trial option order exactly. The config generator is deterministic (fixed internal seed → identical `study-config.json` on re-run).
- **Privacy/no-attribution:** standard repo commit style `type(scope): subject`, GPG-signed, **no AI/Claude attribution** in commit messages.
- **Test baseline:** repo `pytest tests/` stays green (this work adds Python only under `experiments/HS-01/`, no `src/` changes). The app has its own `vitest` + `playwright` suites under `experiments/HS-01/app/`.

**Interfaces:**
- Consumes: `experiments/HS-01/pool_frozen/itempool.json` (frozen, 93 sources / 93 items), `experiments/HS-01/schemas/hs01.study-config.schema.json`.
- Produces: a schema-valid `study-config.json` whose `forms` reference the pool's `item_id`s; a `consent.en.md` with a sha256 recorded in the config; `pool_ref.pool_file_sha256` = sha256 of the frozen itempool file bytes.

**Design (algorithm — implement exactly):**
- Read the pool. Separate items by `kind` (text/image/pair). Find attention items (`is_attention_check == true`): expect exactly `txt-attn-nonsense-01` (text) and `pair-attn-obvious-01` (pair).
- **3 parallel forms** `A`, `B`, `C`. For each phase, take the *regular* (non-attention) items, group by the source's phase-stratum (`sources[source_id].strata[phase]`), sort each group by `item_id` (stable/deterministic), then **deal round-robin** across forms (item index → form `index % 3`). This balances every stratum across forms and partitions each regular item into exactly one form.
- **Attention items go into every form** for their phase (so all raters see both; exclusion threshold = 2 = fail both).
- Assert: within each form, the `source_id`s behind text/image/pair items are pairwise disjoint (holds because each source maps to one phase, but assert anyway); every regular item appears in exactly one form; both attention items appear in all three forms.
- Set the rest of the config from the schema examples (these values are the design defaults — copy verbatim):
  - `scales`: `text-comprehensibility-v1` ("I can tell what this question is asking") and `image-clarity-v1` ("I can tell what this image is displaying"), 5 points, labels `["Strongly disagree","Disagree","Neither agree nor disagree","Agree","Strongly agree"]`.
  - `pair_response`: semantic_options const; display_labels `{OTHER_CLASS:"Something else", NOTHING_RECOGNIZABLE:"Nothing recognizable", CANT_TELL:"I can't tell"}`; `ab_order:"randomized_per_trial"`; `fixed_tail:true`; `other_class_free_text:true`.
  - `phases`: consent 60s; text 120s; image 120s; pair 240s; demographics 60s. `trials_per_rater` for judgment phases = that phase's max per-form item count (informational).
  - `demographics_fields`: age_band / ml_familiarity (3-level + prefer_not_to_say) / english_proficiency (CEFR + native + prefer_not_to_say) / optional free `comment` — exactly as the example.
  - `attention_policy`: `item_ids:["txt-attn-nonsense-01","pair-attn-obvious-01"]`, `placement:"interleaved"`, `exclusion_fail_threshold:2`.
  - `randomization`: `within_phase_shuffle:true`, `session_seed_source:"server_generated"`, `log_seed:true`.
  - `consent`: write `consent.en.md` (neutral framing — see below), set `consent_version:"v1"`, `text_sha256` = sha256 of the file bytes, `required:true`.
  - `quality`: `log_integrity_events:true`, `render_check:true`, `min_rendered_image_css_px:256`.
  - `preregistration`: `{hypotheses_frozen:false, doc_ref:"01 - Active Projects/Master Thesis/Experiments/HS-01-Human-Validity-Study.md"}`.
  - `study_id:"HS-01"`, `config_version:"1.0.0"`, `locale:"en"`, `schema_version:"1.0.0"`.
- **Consent text (`consent.en.md`)** — neutral framing, no mention of "adversarial"/"perturbation"/"testing". Content: purpose ("You will judge short questions and images that an AI system was shown, and tell us how clearly you can understand them."), ~10 minutes, voluntary, anonymous (no personal data, only a participant code), data used for academic research in a master's thesis, you may stop anytime, contact (placeholder `<researcher email>`), a single "I consent and want to begin" affirmation. Keep it short and plain-English.

**Generator code (write this; it is the load-bearing artifact):**

```python
#!/usr/bin/env python3
"""Generate the HS-01 study-config.json (forms + scales + policy) from the frozen pool.
Deterministic: same pool -> identical config. Run via `conda run -n uni python`."""
from __future__ import annotations
import hashlib, json
from pathlib import Path

HERE = Path(__file__).resolve().parent
POOL = HERE / "pool_frozen" / "itempool.json"
OUT_DIR = HERE / "app" / "config"
CONSENT_MD = OUT_DIR / "consent.en.md"
CONFIG_JSON = OUT_DIR / "study-config.json"
N_FORMS = 3
FORM_IDS = ["A", "B", "C"]

CONSENT_TEXT = """\
# Study consent

Thank you for helping with this research.

In this short task you will look at some questions and images that an AI system
was shown, and tell us how clearly **you** can understand them. There are no
right or wrong answers — we are interested in your honest impression.

- It takes about **10 minutes**.
- Participation is **voluntary**; you can stop at any time by closing the tab.
- It is **anonymous**: we store only a random participant code and your answers.
  We do not collect your name, email, or any personal data.
- Your answers are used for academic research in a master's thesis.

Questions: <researcher email>

By choosing **"I consent and want to begin"** you confirm you are at least 18
years old and agree to take part on these terms.
"""

def sha256_bytes(b: bytes) -> str:
    return hashlib.sha256(b).hexdigest()

def main() -> None:
    pool = json.loads(POOL.read_text())
    pool_sha = sha256_bytes(POOL.read_bytes())
    items = pool["items"]
    src_by_id = {s["source_id"]: s for s in pool["sources"]}

    def stratum(it):
        s = src_by_id[it["source_id"]]["strata"]
        return s[it["kind"]] or "_none"

    forms = {f: {"form_id": f, "text_items": [], "image_items": [], "pair_items": []} for f in FORM_IDS}
    key = {"text": "text_items", "image": "image_items", "pair": "pair_items"}
    attention = {it["item_id"]: it for it in items if it.get("is_attention_check")}

    for kind in ("text", "image", "pair"):
        regular = sorted(
            [it for it in items if it["kind"] == kind and not it.get("is_attention_check")],
            key=lambda it: (stratum(it), it["item_id"]),
        )
        # round-robin within stratum: re-sort groups so dealing is stratum-balanced
        for i, it in enumerate(regular):
            forms[FORM_IDS[i % N_FORMS]][key[kind]].append(it["item_id"])
        # attention items -> every form
        for it in items:
            if it["kind"] == kind and it.get("is_attention_check"):
                for f in FORM_IDS:
                    forms[f][key[kind]].append(it["item_id"])

    # invariants
    for f in FORM_IDS:
        srcs = [src_for(items, iid) for iid in forms[f]["text_items"] + forms[f]["image_items"] + forms[f]["pair_items"]]
        assert len(srcs) == len(set(srcs)), f"form {f} reuses a source across phases"
    cover = {}
    for f in FORM_IDS:
        for iid in forms[f]["text_items"] + forms[f]["image_items"] + forms[f]["pair_items"]:
            cover[iid] = cover.get(iid, 0) + 1
    for iid, n in cover.items():
        exp = N_FORMS if iid in attention else 1
        assert n == exp, f"item {iid} appears {n}x, expected {exp}"

    OUT_DIR.mkdir(parents=True, exist_ok=True)
    CONSENT_MD.write_text(CONSENT_TEXT)
    consent_sha = sha256_bytes(CONSENT_MD.read_bytes())

    def per_form_max(k):
        return max(len(forms[f][key[k]]) for f in FORM_IDS)

    config = {
        "schema_version": "1.0.0", "study_id": "HS-01", "config_version": "1.0.0",
        "preregistration": {"hypotheses_frozen": False,
            "doc_ref": "01 - Active Projects/Master Thesis/Experiments/HS-01-Human-Validity-Study.md"},
        "pool_ref": {"pool_id": pool["pool_id"], "pool_file_sha256": pool_sha},
        "locale": "en",
        "phases": [
            {"phase_id": "consent", "target_duration_s": 60, "trials_per_rater": None},
            {"phase_id": "text", "target_duration_s": 120, "trials_per_rater": per_form_max("text")},
            {"phase_id": "image", "target_duration_s": 120, "trials_per_rater": per_form_max("image")},
            {"phase_id": "pair", "target_duration_s": 240, "trials_per_rater": per_form_max("pair")},
            {"phase_id": "demographics", "target_duration_s": 60, "trials_per_rater": None},
        ],
        "scales": [
            {"scale_id": "text-comprehensibility-v1", "applies_to": "text",
             "statement": "I can tell what this question is asking", "points": 5,
             "point_labels": ["Strongly disagree","Disagree","Neither agree nor disagree","Agree","Strongly agree"]},
            {"scale_id": "image-clarity-v1", "applies_to": "image",
             "statement": "I can tell what this image is displaying", "points": 5,
             "point_labels": ["Strongly disagree","Disagree","Neither agree nor disagree","Agree","Strongly agree"]},
        ],
        "pair_response": {
            "semantic_options": ["ANCHOR_WORD","TARGET_WORD","OTHER_CLASS","NOTHING_RECOGNIZABLE","CANT_TELL"],
            "display_labels": {"OTHER_CLASS":"Something else","NOTHING_RECOGNIZABLE":"Nothing recognizable","CANT_TELL":"I can't tell"},
            "ab_order": "randomized_per_trial", "fixed_tail": True, "other_class_free_text": True,
        },
        "forms": [forms[f] for f in FORM_IDS],
        "randomization": {"within_phase_shuffle": True, "session_seed_source": "server_generated", "log_seed": True},
        "attention_policy": {"item_ids": sorted(attention.keys()), "placement": "interleaved", "exclusion_fail_threshold": 2},
        "demographics_fields": [
            {"field_id":"age_band","label":"Your age group","type":"select","options":["18_24","25_34","35_44","45_54","55_plus","prefer_not_to_say"],"required":True},
            {"field_id":"ml_familiarity","label":"How familiar are you with machine learning?","type":"select","options":["no_experience","some_exposure","regular_practice","prefer_not_to_say"],"required":True},
            {"field_id":"english_proficiency","label":"Your English proficiency","type":"select","options":["A1","A2","B1","B2","C1","C2","native","prefer_not_to_say"],"required":True},
            {"field_id":"comment","label":"Anything you want to tell us?","type":"free_text","options":None,"required":False},
        ],
        "consent": {"consent_version": "v1", "text_sha256": consent_sha, "required": True},
        "quality": {"log_integrity_events": True, "render_check": True, "min_rendered_image_css_px": 256},
    }
    CONFIG_JSON.write_text(json.dumps(config, indent=2, ensure_ascii=False) + "\n")
    print(f"wrote {CONFIG_JSON} ({len(items)} items, forms "
          + ", ".join(f'{f}:{len(forms[f]["text_items"])}t/{len(forms[f]["image_items"])}i/{len(forms[f]["pair_items"])}p' for f in FORM_IDS) + ")")

def src_for(items, item_id):
    for it in items:
        if it["item_id"] == item_id:
            return it["source_id"]
    raise KeyError(item_id)

if __name__ == "__main__":
    main()
```

### Task 3: Session storage layer (atomic IO, participant counter, form balancing, validation)

**Files:**
- Create: `src/lib/store.ts`

**Interfaces:**
- Consumes: `loadConfig`, `validateSession`, `HS01_DATA_DIR`.
- Produces:
  - `createSession(channel?): {session_id, participant_code, form, rng_seed, config_version, config_sha256, consent}` — assigns the **least-loaded** form (count sessions whose `status==="completed"` or in-progress modified <24h ago), generates `session_id` (uuid v4), next `participant_code` (`P###` from an atomic counter file), `rng_seed` (random hex), writes the initial session file (see status note), returns the form payload (resolved items with assets) for the client.

  - `submitSession(id, record)` — set `status:"completed"`, `timing.completed_at_utc`/`server_received_at_utc`, atomic write, `validateSession`; persist regardless, return `{ok, errors}`.
  - `statusCounts()` — sessions per form + totals.
- **Status semantics (resolve schema gap):** the session schema `status` enum is only `completed|abandoned`. In-progress files are written with `status:"abandoned"` (a never-submitted session *is* abandoned) and promoted to `completed` on submit. Fresh/partial records that don't yet satisfy `trials minItems:1`/`demographics` are persisted verbatim with `x_validation_errors` (per TOOL-DESIGN "never lose raw data"); strict validity is only expected at submit.

**Details:**
- Atomic write: write `…json.tmp`, `fsync`, `rename`. Counter: read-increment-write the same way; serialize with a tiny in-process mutex (single Node process; n≈30, no external lock needed). Form balancing: list `data/sessions/*.json`, parse `form_id`+`status`+file mtime, count per form (completed OR mtime within 24h), choose the minimum (ties → config form order).

### Task 5: Client timing, seeded randomization, instrumentation, record assembly

**Files:**
- Create: `src/lib/rng.ts`, `src/lib/timing.ts`, `src/lib/instrumentation.ts`, `src/lib/session-record.ts`
- Test: `tests/rng.test.ts`, `tests/timing.test.ts`, `tests/record.test.ts`

**Interfaces (exact):**
- `rng.ts`: `makeRng(seed: string|number): () => number` (mulberry32/xorshift — deterministic); `shuffle<T>(arr, rng): T[]` (Fisher–Yates); `optionOrder(rng): SemanticOption[]` returns the two word options in randomized order followed by the **fixed tail** `[OTHER_CLASS, NOTHING_RECOGNIZABLE, CANT_TELL]`. Determinism: same seed → same sequence.
- `timing.ts`: `class SessionClock` with `started_at_utc`; `nowMs()` returns monotonic offset using `performance.now()`; `rehydrate(load_epoch_utc)` chains epochs across reloads so offsets remain increasing. `markPhase(phase_id, kind:'enter'|'exit')` accumulates `phase_timings`.
- `instrumentation.ts`: `awaitOnset(imgEl?): Promise<{onset_ms, image_loaded_ms|null, rendered_image|null}>` — for image/pair, `await img.decode()` then `requestAnimationFrame` → onset; capture `getBoundingClientRect` (css) and `naturalWidth/Height`; for text, onset after rAF, image fields null. `preloadImage(url)`. `attachIntegrityListeners(clock, sink)` — blur/focus/visibilitychange/resize/fullscreenchange → integrity_events. `renderCheck(): {passed, method}` — canvas-render a known homoglyph string, hash pixels, compare to a baseline ASCII render to confirm non-normalization. `countChanges` helper for `n_changes`.
- `session-record.ts`: `initRecord(createResp, env)`; `appendTrial(record, trial)`; `setDemographics`; `finalizeQualitySummary(record, config)` (computes `attention_total`, `attention_failed` via each attention item's `check_rule` against the stored response, and `focus_loss_count`). Pure functions over the typed record.

**Details:** All `*_ms` from `SessionClock`. `image_loaded_ms` must precede `onset_ms`. The next trial's image is preloaded during the current trial. Stimulus presentation must be **animation-free** (framer-motion only on chrome) so onset is deterministic. `quality_summary` is computed but **exclusion is not applied** (analysis-stage decision per schema).

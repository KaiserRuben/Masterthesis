# BND-005: Embedding Space Alignment (qwen3-embedding)

## Research Questions

1. **Embedding Alignment**: Does embedding `coc_reasoning` with a Qwen3-based model (aligned with Alpamayo's architecture) produce stronger boundary-error correlations than OpenCLIP ViT-bigG/14?
2. **H1 Re-test**: In the aligned embedding space, does `Cor(d_B(z), ADE) < 0` hold?
3. **k-NN Quality**: Do qwen3-space neighbors yield more ADE-predictive single-key-diff pairs?

## Motivation

H1 (boundary-error correlation) was rejected in OpenCLIP space: r = -0.006, p = 0.86.
The hypothesis: the embedding space is **misaligned** -- OpenCLIP ViT-bigG/14 (CLIP-based) vs Alpamayo (Qwen3-based) are fundamentally different representation spaces. Voronoi boundaries in OpenCLIP space don't correspond to Alpamayo's internal decision boundaries.

**Fix**: Embed each scene's `coc_reasoning` text (generated by Alpamayo) using Ollama's `qwen3-embedding`. Since both share the Qwen3 architecture, the embedding space should be aligned with how Alpamayo "understands" scenes.

## Methodology

### Step 1: Embed (`run_embedding.py`)

For each scene with `coc_reasoning` text and `has_ade == True`:

```
1. Load scenes.parquet, filter to scenes with non-empty coc_reasoning AND has_ade
2. Call Ollama embed API in batches (batch_size=32):
     client = Client(host="http://localhost:11434")
     response = client.embed(model="qwen3-embedding", input=batch_texts)
3. L2-normalize all embeddings
4. Save to data/BND-005/qwen3_embeddings.npz
```

### Step 2: Analyze (`run_analysis.py`)

```
A. Load qwen3 embeddings + OpenCLIP embeddings + scenes
B. Re-run H1 (boundary margin d_B vs ADE) in qwen3 space
C. Run same H1 in OpenCLIP space (same scene subset) as baseline
D. Per-key correlation comparison: qwen3 vs OpenCLIP
E. k-NN pair analysis in qwen3 space
F. Save results + comparison figures
```

## Usage

### Prerequisites

```bash
ollama pull qwen3-embedding
```

### 1. Generate Embeddings (requires Ollama running)

```bash
# Default (localhost:11434)
python run_embedding.py

# Custom Ollama host
python run_embedding.py --host http://my-gpu-server:11434
```

### 2. Run Analysis

```bash
python run_analysis.py
```

## Expected Results

| Metric | OpenCLIP (baseline) | Qwen3 (expected) |
|--------|-------------------|-----------------|
| r(d_B, ADE) | -0.006 (n.s.) | Stronger negative |
| Per-key correlations | Weak/mixed | Consistent negative |

If qwen3 shows stronger correlations, it confirms the misalignment hypothesis and validates using architecture-aligned embeddings for boundary analysis.
